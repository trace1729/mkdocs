sql status code

  • 23000 - 违反实体完整性，包含重复条目
  • 21S01 - 插入数据的宽度与数据表的宽度不符合

/getCom

  • 获取数据库所有表单数据

/getMem

  • 通过名称查找 社员

    key = name

/getEvent

  • 通过标题查找 活动

    key = name

/getDept

  • 通过名称查找 部门

    key = name

/insert

插入数据

  • type = { 活动 | 社员 | 类别 | 部门}
  • 社员 - key = {type, 姓名，密码，学院，班级，性别，简介，部门} 完成
  • 部门 - key = {type, time, 名称, 部长编号, 所属社团} 完成
  • 类别 - key = {type, 名称} 完成
  • 活动 - key = {type, 标题, 类别, 内容, 发布者编号, time, 状态} 完成

/del

删除数据

  • type = { 活动 | 社员 | 类别 | 部门 | 附件 | 社团 }
  • key = {type, 编号}
  • return value {error:错误信息, status: 状态码，正常为0，异常时返回sql错误代码, affected_rows：返回删除所影响的行数}

/update

更新数据

  • type = {活动 | 社员 | 类别 | 部门 | 附件 | 社团 }
  • 社员 key = {姓名，密码，学院，班级，性别，简介，部门编号,社员编号}
  • 活动 key = {活动编号，标题，类别编号，内容，附件编号，发布者，发布时间，状态}
  • 社团 key = {社团编号，名称，简介，成立时间，社长编号}
  • 部门 key = {部门编号，名称，简介，社长编号，所属社团}
  • 类别 key = {类别编号，名称}
  • 附件 key = {附件编号，内容，时间，添加人}
